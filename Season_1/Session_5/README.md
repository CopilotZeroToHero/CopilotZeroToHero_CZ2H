# 🚀 CopilotZ2H Session 5: Preparing SharePoint Online: Organising & Governing Content for Copilot

**Session 5 of Copilot: Zero 2 Hero**

## 📝 Overview

This session explores how to organise and govern SharePoint Online content—focusing on site structure, permissions, and data quality—to ensure Microsoft Copilot can access relevant and compliant information.

## ✨ Key Concepts

- **🏢 Flat Site Structure & Hubs:** Create separate sites for discrete topics/teams/projects. Use Hub Sites for unified navigation and search.
- **🏷️ Taxonomy & Managed Metadata:** Standardise tags (e.g., Department, Document Type) to streamline search, reporting, and policy scoping.
- **📄 Content Types & Content Type Hub:** Define required metadata and templates per document type for consistency and compliance.
- **🔐 Permissions & Access Governance:** Manage access via M365/Azure AD groups. Use sensitivity labels and schedule regular access reviews.
- **🛡️ Retention, Labels, and DLP:** Apply retention and sensitivity labels, configure DLP, and archive inactive sites to reduce risk.
- **⚖️ Compliance Alignment:** Align with POPIA, GDPR, and ISO/IEC AI standards for data minimisation, integrity, and accountability.

## 🏡 Homework for Attendees

1. **🔎 Review Your SharePoint Site Structure**  
   List your current SharePoint sites. Are they organised by topic, team, or project? Identify any subsites that could be flattened.

2. **🏷️ Create or Update a Taxonomy**  
   Define a simple term set (e.g., Department, Project) and apply it to a document library using managed metadata columns.

3. **📄 Set Up a Content Type**  
   Create a new content type (e.g., “Project Charter”) with required metadata. Add it to a library and create a sample document.

4. **🔐 Check Permissions**  
   Review who has access to your site/library. Are there any unique permissions? Remove unnecessary access and document your changes.

5. **🛡️ Apply a Retention or Sensitivity Label**  
   Apply a retention or sensitivity label to a document or library. Note how this affects sharing and visibility.

6. **📝 Reflect**  
   Write a short paragraph: How could better information architecture improve Copilot’s answers and your organisation’s compliance?
